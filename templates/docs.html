install

thrum is a 0-dep CLI on mac for now. 

$ curl -sSL https://get.thrum.sh | bash
$ thrum --version

quickstart

Initialize a new project and run your first scan: 

$ thrum init
$ thrum scan .

After the scan, a report appears under out/thrum-report.md.

cli

Currently supported commands:

command description
thrum init create default config + detect project layout
thrum scan . run full detector suite + reasoning + generate report
thrum status show last scan + paths + versions 
$ thrum scan .
→ compiling
→ analyzing
→ running detectors
→ reasoning
→ report saved to out/thrum-report.md

scan output

A scan produces a structured result followed by a human-readable report. 

{
  "project": "dex-v1",
  "critical": 2,
  "medium": 3,
  "info": 2,
  "findings": [
      { "id": 1, "type": "TOCTOU", "severity": "critical", ... },
      { "id": 2, "type": "MV-SI", "severity": "medium", ... }
  ]
}

Reports include code slices, reasoning summaries, and suggested fixes.

detectors

Thrum currently supports a growing suite of semantic security detectors:

  • MV-SCAN (multi-variable SI)
  • TOCTOU & cross-reentrancy
  • upgrade / storage-layout safety
  • oracle and AMM templates
  • ERC / vault correctness
  • access-control & governance

Each detector produces structured facts consumed by the reasoning engine.

ci integration

name: Thrum scan

on:
  pull_request:

jobs:
  thrum:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: curl -sSL https://get.thrum.sh | bash
      - run: thrum scan .

CI scans let you prevent high-severity bugs from ever merging.

faq

  • does thrum store my code?
    currently: no. all analysis happens transiently in memory.
  • can i run thrum offline?
    full detectors require the cloud pipeline. lightweight local analysis coming soon.
  • what languages?
    EVM (solidity) first. others planned.
